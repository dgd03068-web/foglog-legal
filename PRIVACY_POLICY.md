# 포그로그(Foglog) 개인정보처리방침

**최종 수정일**: 2026-05-15  
**시행일**: 2026-05-15

포그로그(이하 "본 서비스" 또는 "앱")는 사용자의 개인정보를 소중히 다루며, 정보통신망 이용촉진 및 정보보호 등에 관한 법률, 개인정보 보호법 등 관련 법령을 준수합니다.

본 방침은 본 서비스가 어떤 정보를 어떻게 수집·이용·보관·파기하는지 투명하게 안내합니다.

## 1. 수집하는 개인정보 항목

본 서비스는 **최소한의 정보**만 수집합니다.

### 1-1. 가입 시
| 항목 | 목적 | 보관 위치 |
|---|---|---|
| 이메일 주소 | 본인 인증 (OTP), 계정 복구 | Supabase Auth (한국 리전) |
| 표시 이름 (선택) | 본인에게만 표시되는 닉네임 | 디바이스 로컬 (SwiftData) |

### 1-2. 서비스 이용 중
| 항목 | 목적 | 보관 위치 |
|---|---|---|
| 일기 본문 (원본) | 본인 보관 | **디바이스 로컬에만** (서버 절대 X) |
| 일기 본문 (익명화본) | 다른 사용자와 공유 (선택 시) | Supabase DB |
| 묶음 토큰 해시 (SHA-256) | 묶음 마감 검증 | Supabase DB |
| 작성 시각·날씨 | 일기 메타데이터 | 디바이스 로컬 |
| 신고 사유 | 부적절 콘텐츠 관리 | Supabase DB |
| 광고 식별자 (IDFA, 동의 시) | AdMob 광고 노출 | AdMob (Google) |

### 1-3. 첨부 데이터 (Phase 1c)
사용자가 일기에 첨부할 수 있는 6가지 데이터의 처리 방침:

| 첨부 | 디바이스 저장 | 익명 공개 시 |
|---|---|---|
| 📷 사진 | Documents/photos/ (EXIF/GPS 자동 제거) | ❌ 비공개 — Phase 2 에서 AI 이미지 단순화 후 공개 예정 |
| 🎤 음성 메모 | Documents/audio/ (m4a) | ❌ 원본 미공개. ✅ 디바이스에서 텍스트로 변환된 전사만 공개 |
| 🎵 음악 | 트랙명/아티스트/앨범 (텍스트만) | ✅ 공개 (식별 위험 낮음) |
| 📍 위치 | 도시명 + 정확 좌표 (GPS) | ⚠️ 도시명만 공개 가능 — 정확 좌표는 디바이스 전용 |
| ❤️ 건강 데이터 | HealthKit 스냅샷 (수면/걸음) | ❌ 전체 미공개 |
| ✅ 체크리스트 | 텍스트 + 완료 여부 | ✅ 공개 (텍스트 익명화 통과) |

### 1-4. **수집하지 않는 정보**
- ❌ 실명 / 주민등록번호 / 휴대전화번호
- ❌ 사진 / 음성 / 건강 데이터의 서버 업로드 (디바이스 전용)
- ❌ 정확한 GPS 좌표의 서버 전송 (도시 단위만)
- ❌ 연락처 / 사진첩 전체 / 카메라 (필요 시점에만 권한 요청, 본 서비스 데이터로 저장 X)
- ❌ 다른 앱 사용 기록
- ❌ 광고 식별자 (사용자가 명시 동의한 경우만)

## 2. 개인정보 이용 목적

1. **본인 인증**: 이메일 OTP 로 가입·로그인
2. **익명 게시판 운영**: 사용자가 명시적으로 "익명 공개" 한 일기만 다른 사용자에게 표시
3. **콘텐츠 관리**: 신고된 부적절 콘텐츠 처리
4. **광고 게재** (선택 동의 시): AdMob 네이티브 광고

본 서비스는 위 목적 외에는 개인정보를 사용하지 않으며, 사용자 동의 없이 제3자에게 제공·판매하지 않습니다.

---

## 3. 익명화의 작동 방식

### 3-1. 익명화 처리
사용자가 일기를 "익명 공개" 할 때:
1. 디바이스에서 사전 + AI 기반으로 인명·지명·회사·학교 등 식별 정보 자동 치환
2. **익명화된 사본**만 서버 전송 (원본은 디바이스에만)
3. 다른 사용자는 익명화된 사본만 열람 가능

### 3-2. 묶음 마감 시
- 묶음 마감 토큰의 해시(SHA-256)가 서버에서 즉시 삭제됨
- 마감된 묶음은 본인도 *어떤 익명 글이 자신의 것인지* 추적 불가
- 즉, **마감 후 영구 익명** 상태가 됨

### 3-3. 익명화의 한계
- 문체(자주 쓰는 말투, 비속어, 띄어쓰기)는 보호되지 않습니다
- 사건의 고유성(다른 사람은 모르는 구체적 사건)도 보호되지 않습니다
- 정보기관급 추적은 막을 수 없습니다

## 5. 안전 조치

- **HTTPS / TLS 1.3** 강제: 모든 통신 암호화
- **이중 격리**: Anthropic API 키 등 민감 정보는 Supabase Edge Function 내부에만 존재, 앱에 절대 노출 X
- **Row Level Security (RLS)**: 데이터베이스 레벨에서 권한 검증
- **로컬 암호화**: 디바이스 키체인(Keychain) 으로 묶음 토큰 보호 (Whenever Unlocked, This Device Only)

## 6. 사용자 권리

본인은 언제든 다음 권리를 행사할 수 있습니다:

1. **개인정보 열람 / 수정**: 앱 내 "내정보" 화면
2. **회원 탈퇴**: "내정보 → 계정 → 계정 삭제"
3. **익명 공개 묶음 마감**: "내정보 → 활성 묶음 → 마무리"
4. **광고 식별자 거부**: iOS 설정 → 개인정보 보호 → 추적

**탈퇴 효과**:
- 디바이스 로컬의 모든 일기·로그 영구 삭제
- Supabase 이메일·세션 제거
- ⚠️ **이미 익명 공개된 묶음은 자동 삭제되지 않습니다** — 사전에 본인이 마감하시기 바랍니다 (이미 익명화돼 식별 불가능하므로)
